## Capítulo 1 Aula 4 – Como a Internet chega na minha casa?

História da Internet:

A Internet nasceu durante o perído da Guerra Fria (1949). Em 1969 a Rússia lançou o primeiro satélite (Sputnik). No mesmo ano o presidente dos EUA, Dwight Eisenhower fundou a DARPA, uma agência para estudos de tecnologias para guerra. Em 1969 a DARPA criou a ARPANET, uma rede interna para fazer um backup dos dados dos centros militares. A universidade da Califórnia criou o protocolo de controle de rede (NCP).  Em 1972 o Bob Kahn criou o TCP para melhorar a comunicação da rede que estava crescendo. No mesmo ano, o vint Cerf criou o IP. Juntando o trabalho dos dois, foi criado o protocolo TCP/IP, que usamos até hoje.

A ARPANET cresceu tanto que os militares resolveram dividir e criaram a MILNET (rede apenas militar), NSFNET (rede das universidades) e redes comerciais. 

Rede mundial → [www.submarinecablemap.com](http://www.submarinecablemap.com/)

Internet é um conjunto de redes.

Como funciona a Internet → https://youtube/TNQsmPf24go

Como era a Internet?

Tela preta com letras brancas ou amarelas → gopher

Em 1993, o inglês **TimBerners-Lee** criou textos que poderiam ser acessados através de link’s (hiper-links), nasce o protocolo **http://.** Ele também criou o HTML</> (Linguagem de marcação para hiper textos). O **Marc Andreessen** criou o primeiro navegador da www, o **Mosaic.**

World Wide Web → rede de alcance mundial.

## Capítulo 2 Aula 1 – Como a Internet funciona?

**Representação de dados** → sinais (0 e 1) binary digit – bit (dígitos dbináios).

01000001 → Oito bits, é a porção mínima para se represente um dado, também chamada de byte.

Código multibyte UTF-8.

1024 bytes = 1 KB

1024 KG = 1MB

1024 MB = 1 GB

1024 GB = 1 TB

1024 TB = 1 PB

1024 PB = 1 EB

1024 EB = 1 ZB

1024 ZB = 1 YB

**OBS.:**

MB != Bb

MB → MEGABYTES, ultilizado mais para armazenamento.

Mb → MEGABITES, ultilizado mais para transmissão.

**Como nos conectamos?**

Modem → aparelho que faz a modulação (transformação das ondas quadradas para ondas senoidal) e demodulação (transformação das ondas senoidal para quadradas). O modem permite a comunicação entre o computador (cliente) e a Internet.

DNS → Sistema de nome de domínio – local na Internet, servidores especiais, onde será pesquisado o endereço IP do domínio que eu quero acessar. Com esse endreço IP, meu computador pode encontrar o domnínio digitado no navegador.

**E as rotas?**

Um solicitação de acesso a um site, pode ir por várias rotas diferentes em cada solicitação. Por isso, o carregamento do site pode ser mais rápido ou mais lento.

## Capítulo 2 Aula 2 – O que é domínio e hospedagem?

**Domínio** → é o nome que identifica o seu site. Nome único, pago anualmente, vários TLDs.

**Hospedagem** → é o local aonde seu site vai estar armazenado. Espaço para armazenar arquivos. Pago mensalmente. Escolhe por espaço, memória, recursos.

URL → Uniform Resource Locator (Localizador de Recurso Único/Uniforme), ex.:

**www.github.com/gustavoguanabara**

Domínio → www.**github.com**/gustavoguanabara 

Nome do domínio → www.**github**.com/gustavoguanabara 

TLD → www.github**.com**/gustavoguanabara 

Sub-domínio → **www**.github.com/gustavoguanabara 

Caminho → www.github.com**/gustavoguanabara**

## Capítulo 3 Aula 2 – Front-end, Back-end e Full stack

HTML, CSS e JavaScript são consideradas tecnologias client-side, ou seja, não vão funcionar diretamente no servidor, mas no navegador de sua máquina. O desenvolvedor dessas tecnologias é chamado de desenvolvedor **Client-side** ou **Front-end**.

**Server-side** são tecnologias que funcionam diretamente no servidor. O desenvolvedor dessas tecnologias é chamado de desenvolvedor **Back-end**.

![img](file:///tmp/lu11266z2kqow.tmp/lu11266z2kqpf_tmp_39b97cf52f38c062.png) 

O desenvolvedor que tem domínio nas duas áreas, **Front-end** e **Back-end** é chamado desenvolvedor **Full Stack.** Geralmente este desenvolvedor é um profissional experiente e demora um bom tempo para chegar a este nível.

1. **Capítulo 12 Aula 1 – Estilos CSS inline**

Estilos inline → em cada tag eu consigo colocar o estilo em CSS.



<body style="background-color: lightblue;">

​	<h1 style="background-color: lightcoral; color: blue; font-family: Courier New">Capítulo 1</h1>

​	<h2 style="color: red; ">Capítulo 1.1</h2>

	<p style="text-align: justify;">Lorem ipsum dolor sit amet consectetur adipisicing elit. Ratione doloremque impedit asperiores nobis! Fuga provident, eveniet laudantium, facere eligendi esse doloribus est dolorum similique ipsum ut porro quam quibusdam odio.</p>



A desvatagem desse tipo de estilização é que terá que ser aplicado o estilo em cada tag de todo o HTML. Isso requer mais trabalho e polui o HTML.



1. **Capítulo 12 Aula 2 – Estilos CSS internos**

CSS interna ou CSS local → Será adicionada a tag <style></style> sdfno “head” do seu documento HTML e dentro da tag <style></style> será construído o estilo do seu site:



<style>

​	body{

​		background-color: lightblue;

​		font-family: courier;

​		font-size: 20px;

​	}

​	h1{

​		color:darkblue;

​		background-color:lightcoral;

​	}

​	h2{

​		color: darkred;

​	}

​	p{

​		text-align: justify;

​	}

</style>



Uma desvantagem em usar esse tipo é que ele ocuparaá muitas linha em seu arquivo HTML e outro é que caso você tenha muitas página e queira adicionar o mesmo estilo em todas, terá que colocar a tag <style></style> em cada página HTML.



1. **Capítulo 12 Aula 3 – Estilos CSS externos**

Folhas de estilo CSS

Um <link rel="stylesheet" href="style.css"> será colocado no <head></head> no seu arquivo HTML para fazer a ligação para um arquivo separado que conterá apenas o estilo CSS. Esse link pode ser colocado em qualquer página do seu site, logo todas as páginas terão o mesmo estilo. No arquivo style.css será colocado o estilo da página do seu site.



body{

​	background-color: lightgoldenrodyellow;

​	font-family: courier;

​	font-size: 20px;

}

h1{

​	color: maroon;

​	background-color:darkgoldenrod;

}

h2{

​	color: darkred;

}

p{

​	text-align: justify;

}



No mesmo arquivo pode ser usado o estilo inline, interno e externo.
